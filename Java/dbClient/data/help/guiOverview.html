GUI Overview

This tool's GUI is composed of three main area which are:
  • Menu Area
  • Left Area
  • Right Top Area
  • Right Bottom Area

Fig 1. Overall view.

Menu Area

The Menu Area is composed of the menu for this application.

Fig 2. Menu.
  • File: Contains some general options. Options:
    • Quit: Closes the application.
  • View: Allows to activate or deactivate some of the available views. Options:
    • Log View: Allows to hide or display the Log View.
  • Actions: Contains the actions available to the current selected element on the tree view (Left Area).
  • Help: Contains a link to this help. Options:
    • About: Simple view of this tool identification and version.
    • Content: This help.

Left Area

This area is composed of a tree constituted of nodes which correspond to connections, buckets and tuples. To access the actions that can be executed for an node select a node and now there are two ways to see the actions available by:

  • Double click on the node which will show a popupmenu with the available action for that specific node.
  • Select the main manu "Actions" to show the popupmenu with the available action for that specific node.

Once the popupmenu is shown select the appropriate active option to execute the corresponding operation (action).


The popup have some actions specific for the type of node, so they are common to all the NOSQL databases and others specific to the current database(s) connected to, which appear at the bottom of the popupmenu.


The main node type are:

Fig 3. Tree view.
  • Root: root of the system that allows to create multiple connections.
  • Connection: a connection that when created will ne to be executed the connect action before being able to create or read buckets.
  • Bucket: a bucket containing none or many tuples.
  • Tuple: composed of a key and a value.

Right Top Area

On this area it is displayed the details of the details for the selection made on the Left Area. On top of this is displayed the application menu in some OSs and the title for this application. In Mac the menu is displayed on the top of the screen.

Fig 4. Details view.

Right Bottom Area

On tis area the logged messages are displayed and the log menu is shown too. The log view area is at the bottom and on top it is the log menu.

Fig 5. Log view.
  •   it is close button. When pressed the full log view is hidden. To show the full log view go to the menu under View.
  •   it is clear button. When pressed the log view is cleared of any message.
  •   it is help button. When pressed the simple command line help is displayed on the log view.
  • The combo list allow to filter the message types displayed on the log view. The message types are displayed in ascending oder of priority. Message type lower in the combo list corresponds to higher priority. Any selection identify from which priority messages will be displayed on the log view. The available type:
    • ALL: shows all the messages.
    • FINEST: shows all the messages from those of type 'FINEST' to 'SEVERE'and with no-type.
    • FINER: shows all the messages from those of type 'FINER' to 'SEVERE' and with no-type.
    • FINE: shows all the messages from those of type 'FINE' to 'SEVERE' and with no-type.
    • CONFIG: shows all the messages from those of type 'CONFIG' to 'SEVERE' and with no-type.
    • INFO: shows all the messages from those of type 'INFO' to 'SEVERE' and with no-type.
    • WARNING: shows all the messages from those of type 'WARNING' to 'SEVERE' and with no-type.
    • SEVERE: shows all the messages from those of type 'SEVERE' and with no-type.
    • OFF: shows all the messages from those with no-type.
